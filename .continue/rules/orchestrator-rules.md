# Engineering Orchestration Rules

The orchestrator coordinates all agents following this workflow:

1. Analyst gathers requirements
2. PM structures tasks
3. Architect defines system design
4. Coder implements the feature
5. Tester generates unit and integration tests
6. Reviewer evaluates code quality
7. Security agent analyzes vulnerabilities
8. Performance agent evaluates performance
9. Memory agent stores insights for future tasks

Rules:

- Always follow docs/AI_CONTEXT.md
- Prefer modifying existing modules instead of creating new ones
- Avoid architectural drift
- Ensure tests exist before merging new code
